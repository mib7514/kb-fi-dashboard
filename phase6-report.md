## 1. 채점 지표 (§1 RG-4)

| 지표 | 정의 | 코드 |
|---|---|---|
| 최빈 셀 적중 | judgment.mode.cell == 실현 셀 | `modalHit` |
| 축별 적중 | argmax(금리 확률)/argmax(스프레드 확률) == 실현 방향 | `axisHit.{rate,spread}` |
| 9셀 Brier | Σ(p−o)², p=P(금리)×P(스프레드) 9셀 분수, o=실현 원핫 | `brier.cells9` |
| 섹터 Brier | 섹터별 3분류 Σ(p−o)² (6섹터) | `brier.sectors.perSector` |
| RG-2 순위 | 선택 구간이 실현 최고수익 top-1/top-2 인가 | `rg2Rank.{hitTop1,hitTop2}` |

- 9셀 분포는 저장하지 않고 `judgment.probs.rate × spread`로 재구성(파생, 무손실).
- RG-2 실현수익 = `carryRollBp[k] − D′[k]·실현Δy[k]` (캐리+롤다운은 판단 시점 고정, 커브이동만 실현으로 대체).
  `carryRollBp`는 판단 확정 시 저장한 구간별 캐리+롤다운(파생 bp, 레벨 아님).

## 2. 분류 규칙 — 경계 처리 (체크리스트 ②)

```
classifyRealized(Δ, band):  Δ ≥ +band → 상승/확대 · Δ ≤ −band → 하락/축소 · 그 외(|Δ|<band) → 보합
```
- **등호(=밴드)는 방향으로 처리** — Phase 1 `calibrate.mjs`의 셀 분류(`Δ>=band → up`)와 동일 규칙 →
  입력·채점·캘리브레이션 삼자 일관.
- 예(국고3Y, band ±4.6): Δ=+4.5 → 보합, Δ=+4.6 → 상승, Δ=+4.7 → 상승, Δ=−4.6 → 하락.
  (체크리스트 ② "밴드−0.1 → 보합, +0.1 → 방향" 충족, 등호는 방향.)

## 3. Brier 손계산 검증 (체크리스트 ③)

**판단:** 금리[60/30/10]% 스프레드[20/30/50]% → 정규화 [.6/.3/.1]×[.2/.3/.5].
9셀 분수: down|narrow .12 · down|flat .18 · **down|wide .30** · flat|narrow .06 · flat|flat .09 ·
flat|wide .15 · up|narrow .02 · up|flat .03 · up|wide .05 (합 1.00).

**실현:** 국고3Y Δ=−10(≤−4.6 → down) · 대표스프레드 Δ=+8(≥2.2 → wide) → 실현 셀 **down|wide**.

**Brier = Σ(p−o)²** (o 원핫 at down|wide):
```
= .12² + .18² + (.30−1)² + .06² + .09² + .15² + .02² + .03² + .05²
= .0144 + .0324 + .49 + .0036 + .0081 + .0225 + .0004 + .0009 + .0025
= 0.5748  → round2 = 0.57
```
**코드 결과 `brier.cells9 = 0.57` = 손계산.** (단위테스트 `brierMulti(...) = 0.5748` 정확 일치.)

섹터 예(공사채 [50/30/20]%, 실현 Δ=−3 ≤ −1.9 → narrow):
Brier = (.5−1)² + .3² + .2² = 0.25+0.09+0.04 = **0.38** = 코드.

## 4. 공유 2행 이중계산 처리 (체크리스트 ⑥)

**판단·근거:**
- 국고채 섹터는 밴드·실현 소스가 **금리 축(ktb3y)과 동일 신호**, 회사채 섹터는 **스프레드 축(repSpread)과
  동일 신호**다(Phase 5 상태 공유의 귀결). 두 섹터의 3분류 Brier는 축 채점·9셀 Brier와 수학적으로 중복된다.
- **처리 = "산출하되 헤드라인 집계에서 제외"**:
  - `perSector`에 6섹터 Brier 전부 기록(투명성 — 대시보드 섹터 표에 6열 표시).
  - 헤드라인 집계 `brier.sectors.creditAvg`는 **비공유 신용 4섹터(공사/은행/카드/여전)만** 평균 →
    축·9셀에 이미 반영된 금리·스프레드 신호를 이중가중하지 않음.
  - 참고용 `allAvg`(6섹터)도 병기. 각 `perSector[국고채].shared='rate'`, `[회사채].shared='spread'` 플래그.
- **근거:** 섹터 보드의 독립 정보는 신용 4섹터에 있고, 국고·회사는 축 지표로 이미 채점된다. 완전 제외(미산출)
  대신 산출+집계제외를 택한 이유는 사용자가 6섹터 전체를 한눈에 보되 요약치는 편향 없이 얻게 하기 위함.

## 5. UI · 운영 규칙 (§4)

- **미결 창:** `judgments` 중 `scores` 없는 주차, 만기(判+1개월)·D-day 표시, 만기순 정렬, 도래분 강조.
  중첩 4~5개 동시 표시. 채점 select 에는 만기 도래분만 노출.
- **채점 입력:** 만기 도래 창 선택 → 실현값(국고3Y Δ·대표 스프레드 Δ·신용 4섹터 Δ·커브 8구간 Δ, 전부
  파생 Δbp) → 자동 분류·채점 → `scores` append 스니펫 + 복사. **커브 레벨 원본은 받지 않음(§0.3).**
- **누적 대시보드:** 최빈·축별 적중률, Brier 9셀/섹터 평균, 주차별 Brier 추이 미니바, 섹터별 평균 Brier 표.
  **2건 미만 → "표본 부족" 안내.**
- **운영 규칙 표기(UI):** "채점 완료 주차의 judgment/score는 수정하지 않습니다(관례). 오입력 시 scores 항목
  삭제 후 재채점." (코드 강제 아님 — 원장 운영 규칙, §4.)
