데이터 업로드

KOSIS CSV → 파싱 → 전망 미리보기 → data/*.js 생성. 다운로드한 파일을 레포에 커밋하면 조회 페이지에 반영됩니다.

1. 시리즈 선택

이 CSV로 갱신할 시리즈. 선택하면 CSV에서 해당 행을 자동 매칭합니다.

2. CSV 업로드

KOSIS 소비자물가지수 원본 CSV (wide 형식, 총지수 행 포함).

3. 데이터 행 확인

CSV에서 감지된 행. 원하는 시리즈에 맞는 행을 선택하세요 (자동 매칭됨).

4. 전망 미리보기

선택한 데이터로 calc 엔진이 계산한 결과. 이상 없으면 export.

5. 발행

data/*.js 파일을 생성해 다운로드합니다.

흐름: 다운로드 → data/ 폴더에 넣기 → GitHub 커밋 → 조회 페이지 자동 반영

크레딧 스프레드 (composite xlsx)

composite xlsx 업로드 → 파싱·G1 검증 → data/credit-spread.js 생성. 파서는 tools/convert-composite.mjs 와 동일 모듈을 써 산출물이 바이트 동일합니다.

xlsx 업로드

credit_spread_composite_영업일만.xlsx — 시트 spread, 헤더 17행. 국고=금리%, 크레딧=스프레드%p.

검증 미리보기

G1 특성화 게이트 통과 시 최신일자·행수·공사AAA 3년 현재 %ile 을 표시합니다.

발행

data/credit-spread.js 를 생성해 다운로드합니다 (node 스크립트 산출물과 바이트 동일).

흐름: 다운로드 → data/ 폴더에 넣기 → GitHub 커밋 → Curve RV 페이지 자동 반영

On/Off 스프레드 (세대별 민평 xlsx)

인포맥스 종목별 민평 xlsx 업로드 → 세대 정렬 + 커브조정 스프레드(bp) 변환 → data/onoff-ktb3y.js 생성. 원본 수익률은 브라우저 메모리에만 있고, 산출물에는 파생 스프레드만 담깁니다(유료 벤더 데이터 재배포 금지).

xlsx 업로드

표ktb3y_지표비지표.xlsx — 시트 Sheet1, row1 종목명(2열 간격), row3~ [일자·수익률]. 주말 행은 자동 제거됩니다.

세대별 변환 미리보기

구조 검증 통과 시 세대 수·현재 세대 분해(raw/slope/fly)·세대 목록을 표시합니다.

발행

data/onoff-ktb3y.js 를 생성해 다운로드합니다 (node 스크립트 산출물과 바이트 동일).

흐름: 다운로드 → data/ 폴더에 넣기 → GitHub 커밋 → 조회 페이지 자동 반영

입찰일 이벤트

국고 3년 입찰일을 일괄 입력 → data/onoff-events.js 생성. 분기·반기말은 자동 계산되므로 입력 불필요. 판정 엔진의 컨세션 룰에 사용됩니다.

입찰일 입력

쉼표·공백·줄바꿈 구분. 예: 2026-07-06, 2026-08-03, 2026-09-07

코멘터리

조회 페이지의 [메트릭+판정 JSON 복사] 결과를 붙여넣고 코멘터리를 작성 → 기존 히스토리에 append 하여 data/onoff-commentary.js 생성.

1. JSON 스냅샷 붙여넣기

조회 페이지에서 복사한 { fly/z/분해, 최근 20영업일, 판정+evidence, 이벤트 } JSON.

2. 코멘터리 텍스트

현재 로드된 히스토리 수: 0건 — 다운로드는 여기에 1건 append.
