# 디자인 원칙 (fi-dashboard 공용)

측정/해석 분리·무빌드 정적·다크 테마 등 데이터·코드 규약은 각 모듈 파일 상단 주석과
`js/nav.js`(공통 UI 단일 근원)에 있다. 이 문서는 **화면(차트·표) 공용 원칙**을 모은다.

## 차트

- **끝점 값 라벨 필수.** 모든 시계열 차트의 마지막 점(최신 실측·전망 끝)에는 값 라벨을
  직접 표기한다. 눈-왕복(축↔점) 없이 **차트 단독으로 읽혀야** 한다.
  - 값은 데이터에서 동적 바인딩(하드코딩 금지).
  - 라벨이 여럿이면 세로 최소 간격을 보장하고, 겹치면 지시선(leader)으로 분리한다.
  - 기준점(마지막 실측 등)이 있으면 "여기까지 실제" 류의 앵커 문구로 명시한다.
  - 적용 예: `us-inflation.html`(유가 밴드 3갈래+실측 앵커), `inflation-diffusion.html`
    (ge2 실선·ex-E 점선 끝점) — 신규 시계열 차트도 동일 규칙을 따른다.

## 발표용 내보내기 프리셋

슬라이드 삽입용 고해상도 PNG는 화면 캡처가 아니라 **내보내기 해상도를 직접 지정**한다.
근거: 슬라이드의 차트 칸이 약 2.7인치인데, 세로 모드 380px 타일을 그대로 캡처하면
2.7인치 기준 약 140dpi로 해상도·글자 크기가 모두 미달한다.

구현은 `js/curve-phase/cp-charts.js`의 `exportChart(elId, filename)` — 화면 차트는
건드리지 않고(딥카피) 화면 밖 숨김 컨테이너에 프리셋을 적용해 렌더 후 다운로드한다.

| 항목 | 값 |
| --- | --- |
| 캔버스 | 1200 × 1450 px (세로 380:460 비율 유지) |
| Plotly scale | 2 → 최종 **2400 × 2900 px** (2.7인치 기준 약 890dpi, 요구 400dpi 상회) |
| 배율 상수 M | 1200 / 380 ≈ 3.16 |
| 폰트 | 세로 모드 각 폰트값 × M × 1.5 (겉보기 1.5배) |
| 선 굵기 | 기존 × M × 2 (겉보기 2배 — 사이클 오버레이 current 선 포함) |
| 마커 | 기존 × M × 1.5 |
| 그리드 | `#f0ede6` (light 팔레트보다 한 단계 옅게), zeroline `#e2ddd2` 동일 계열 |
| 팔레트 | 화면 테마와 무관하게 **항상 light 고정** |
| 배경 | `#ffffff` 불투명 (투명 금지 — 슬라이드 삽입용) |
| 파일명 | `cp-{차트id}_present.png` |

- 대상은 **플롯 카드만**(`.chart-box` 보유). 게이지·표 카드는 제외.
- 후속 차트(슬로프 탭 등)도 이 프리셋을 재사용한다 — 차트별 상수를 새로 만들지 말고
  `exportChart`/`buildExportSpec` 경로에 태운다.

## 쉬운 화면

- 측정 레이어 화면은 전문용어 노출을 피하고 쉬운 라벨을 쓴다(예: "우리 계산기",
  "계산기+기름값"). 해석·전략 코멘트는 하드코딩하지 않는다.
