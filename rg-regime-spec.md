## 0. 목적과 경계

### 0.1 목적

과거 경로 기반(후행) 레짐 판정이 아니라, **확률적 미래 전망과 현재 수준의 gap**으로
"현재부터 1개월 후까지의 레짐"을 전향적으로 판단한다. 주간 단위로 판단을 기록하고,
1개월 후 실현값과 대조하여 채점한다 — 다음 판단 개선을 위한 피드백 루프.

### 0.2 bpbybp / Fenrir 경계

| 항목 | bpbybp (본 모듈) | Fenrir |
|---|---|---|
| 입력 | 확률·기대 Δ **숫자만** | 시나리오 논거, 분지도, 레벨 상세 |
| 플레이북 | 이론적·교과서적 정석 (전문 공개) | Z/S/C/L 버킷 매핑, 실전 전략 |
| 저장 | 파생값만 (확률, Δbp, 채점 결과) | 판단 원장 원본, 해석 |

- 근거(why)는 절대 bpbybp에 기록하지 않는다. 무엇을(what) 예상하는가만.
- Z/S/C/L 버킷 언급, 레짐별 배분 테이블, 레버리지 가이드는 **포함 금지**.

### 0.3 데이터 라이선스 원칙

- 민평 수익률 **원본은 저장하지 않는다**. 세션 내 계산용 입력으로만 사용.
- 저장 가능: 구간별 캐리+롤다운(파생 bp), 커브이동 기대 Δbp, 확률, 채점 결과,
  캘리브레이션 결과 테이블(레짐별 중위 Δ — 파생 통계값).
- on/off 모듈(OO)과 동일 원칙.
- **개정(2026-07, Taylor 모듈):** 공개 API(ECOS·FRED 등) 출처 시계열은 **금리 레벨 저장 허용**.
  단말(Infomax 등) 유래 데이터는 기존 원칙(Δbp 파생만 저장) 유지.

## 1. 모듈 정의

### RG-1 — 전향적 레짐 히트맵

**입력 (주 1회, 수동):**
- 금리 방향 확률 3개: P(하락), P(보합), P(상승) — 국고3Y 기준, 1개월 호라이즌
- 스프레드 방향 확률 3개: P(축소), P(보합), P(확대) — 회사채 AA- 3Y − 국고3Y 기준
- 총 6개 숫자. 숫자 입력 + 슬라이더 병행.

**확률 입력 UX (전 모듈 공통):**
- 수동 입력. 슬라이더 조작 시 다른 확률 자동 조정 없음.
- 축별 합계 ≠ 100% 시 경고 뱃지 + 원클릭 정규화 버튼 (CB 모듈 p-정규화 패턴 재사용).
- 정규화 전에는 저장 불가.

**출력:**
- 9셀 결합확률 = P(금리 i) × P(스프레드 j) — 축 간 독립 가정 (v1 명시적 한계, UI에 주석)
- 3×3 히트맵 (확률 → 색 농도), 최빈 셀 하이라이트 + 국면 명칭 표시
- 최빈 셀 확률, 상위 2셀 확률 합계 표시 (확신도 대용 지표)

**플레이북 전문 (히트맵 하단 상시 전개, 접기 없음):**

| 셀 | 국면 명칭 | 정석 플레이북 |
|---|---|---|
| ↓/축소 | 리스크온 랠리 | 듀레이션 오버웨이트 + 장기 크레딧. 금리 하락과 스프레드 압축 동시 수취. 레버리지 캐리 유효성 최고 구간 |
| ↓/보합 | 듀레이션 장세 | 국채 중심 듀레이션 확대, 크레딧 중립. 단기금리 주도면 스티프너, 장기 주도면 롱엔드 집중 |
| ↓/확대 | 리스크오프 | 국채 듀레이션 유지·확대, 크레딧 언더웨이트, 보유분 우량 전환(quality up). 유동성 확보 우선 |
| →/축소 | 캐리 장세 | 크레딧 오버웨이트로 캐리&롤다운 극대화. 레버리지 캐리 손익분기 여유 최대 구간 |
| →/보합 | 중립·정체 | 방향 베팅 자제, 벤치마크 근접. 유동성·신규발행 프리미엄 등 마이크로 알파 수확, 옵션성 축적 |
| →/확대 | 크레딧 경계 | 신용사이클 악화 신호. 크레딧 축소 + 등급 상향, 부실 시그널 모니터링 격상 |
| ↑/축소 | 리플레이션 | 성장 주도 금리 상승. 듀레이션 언더웨이트, 단기 크레딧 캐리로 스프레드 압축 수취 |
| ↑/보합 | 베어 금리 | 듀레이션 축소, 단기물·FRN 성격 자산 회전, 현금흐름 재투자 대기 |
| ↑/확대 | 이중 약세 | 듀레이션·크레딧 동시 축소, 최단기·현금 중심 전면 방어. 다음 레짐 전환 진입 준비 국면 |

- 색상 인코딩: 우호(리스크 확장) / 중립(선별 대응) / 방어(리스크 축소) 3단 + 범례.

### RG-2 — 롤다운 탐색기

**수익 분해 (1개월 보유, 구간별):**

```
기대수익(bp) = 캐리 + 롤다운 + 커브이동 효과
  캐리      = y0 × (1/12)                      … 현재 수익률로 확정, 전망 무관
  롤다운    = −D × (y_현재커브(t−1M) − y0)      … 착지커브 = 현재커브 가정
  커브이동  = −D × E[Δy_구간]                   … v1/v2 차이는 이 성분만
```

- 구간 격자: 3M / 6M / 1Y / 1.5Y / 2Y / 2.5Y / 3Y / 5Y (8구간, Z·S·C·L 경계 정렬)
- 커브 입력: 구간별 민평 수익률 수동 입력 (세션 계산용, 비저장 — §0.3)
- 듀레이션 D: 구간 만기 기반 근사 (v1: 수정듀레이션 ≈ 잔존만기, 쿠폰 근사는 v2 검토)

**v1 — 평행이동:**
- E[Δy] = RG-1 금리 시나리오의 확률가중 기대값 (추가 입력 없음, RG-1 재사용)
- 전 구간 동일 적용. UI에 "평행이동 가정" 뱃지.

**v2 — 시나리오별 구간 Δ (24칸):**
- 시나리오 3개(하락/보합/상승) × 구간 8개 = 24칸. 숫자 + 슬라이더 병행.
- 시나리오 3개를 탭 아님, **한 화면 세로 전개** (스트리밍·한눈 비교).
- 기본값 2층 구조:
  - **1층 — 9레짐 조건부 역사 기본값**:
    `하락 시나리오 기본커브 = Σj P(스프레드 j) × 중위Δ커브[레짐(하락, j)]`
    RG-1 스프레드 확률로 3셀을 확률가중. RG-1 입력 변경 시 미수정 칸의 기본값
    실시간 갱신 (수정된 칸은 유지).
  - **2층 — 전주 이월**: 신규 주간 입력 시 시작값 = 전주 확정값.
    "역사 기본값으로 리셋" 버튼 상시 제공.
  - 보합 시나리오 1층 기본값 = 전 구간 0.
- 수정된 칸은 기본값 대비 변경 표시(dot indicator) — "어디에 뷰를 실었는가" 가시화.
- 확률가중: E[Δy_구간] = Σi P(금리 i) × Δ커브[i][구간]

**혼합 뷰:**
- 커브이동 성분만 `w × 평행(v1) + (1−w) × 내전망(v2)`, w 슬라이더 (0~100%)
- 해석: 무정보 prior(평행) 대비 내 뷰의 shrinkage 비중.

**출력:**
- 구간별 기대수익 막대 (3성분 스택), v1/v2/혼합 전환, 최고 기대수익 구간 하이라이트
- 순위 테이블 (기대수익 내림차순)

### RG-3 — 섹터 스프레드 보드

**대상 6섹터:** 국고채 / 공사채 / 은행채 / 회사채 / 카드채 / 여전채

**입력 (간단형, 주 1회):**
- 섹터별 방향 확률 3개 (축소/보합/확대) × 6섹터 = 18칸. 숫자 + 슬라이더.
- 기대 Δ 근사: E[Δs] ≈ P(축소)×(−δ) + P(확대)×(+δ), δ = 해당 섹터 보합밴드 경계값
- 정밀형(시나리오별 대표 Δbp 입력) 승격은 운용 후 판단 — §6 보류 항목.

**출력:**
- 섹터별 방향 확률 바 + 기대 Δ 순위
- 회사채 행은 RG-1 스프레드 축과 동일 입력 공유 (이중 입력 방지 — RG-1에서 입력하면
  RG-3 회사채 행에 자동 반영, 역방향 동일)

### RG-4 — 채점 원장

**기록 (주간 판단 확정 시):**
- 판단일, 만기일(판단일 + 1개월), RG-1 확률 6개 + 9셀 분포, RG-2 선택 최상위 구간,
  RG-3 섹터별 확률, 기준 현재값(국고3Y, 대표 스프레드, 섹터 스프레드 — Δ 산출용 파생)
- 1개월 호라이즌 + 주간 기록 → 미결 창 항상 최대 4~5개 중첩. 만기 도래 순 채점.

**채점 (만기 도래 시, 수동 트리거 + 실현값 입력):**
- 실현 방향 분류: σ 보합밴드 적용 (|Δ실현| < 밴드 → 보합). **입력 시와 동일 기준.**
- 지표:
  1. RG-1 최빈 셀 적중 (hit/miss)
  2. 축별 방향 적중 (금리 / 스프레드 각각)
  3. Brier score: RG-1 9셀 분포 vs 실현 셀 (multi-class), 섹터별 3분류 Brier
  4. RG-2 순위 적중: 선택 구간이 실현 최고수익 구간이었는가 (top-1 / top-2 이내)
- 누적 대시보드: 최근 N회 적중률 추이, Brier 추이 (캘리브레이션 개선 여부),
  축별·섹터별 분해.

## 2. 판정 기준 — σ 보합밴드

- 각 계열(국고3Y, 대표 스프레드, 섹터 스프레드 6종)의 **1개월 변화 역사 분포** 산출
- 보합밴드 = ±k×σ, 초기값 k = 0.25 (캘리브레이션 단계에서 실측 후 확정,
  대략 금리 ±4~5bp 수준 예상)
- 밴드 값은 계열별 상수로 캘리브레이션 결과 테이블에 저장, UI에 명시
- 입력 가이드(사용자가 확률 배분 시 참조)와 채점 분류에 **동일 값** 적용

## 3. 캘리브레이션 (오프라인 1회 작업, Phase 1)

**데이터:** Infomax 민평 국고3Y + 섹터별 스프레드, 가용 최장 기간 (목표 15~20년)

**산출물 (repo 저장은 결과 파생값만, 단일 파일):**
- `data/rg-calib.js` → `window.RG_CALIB = { bands: {...}, medianCurves: {...}, meta: {...} }`
  - `bands`: 계열별 σ, 보합밴드 (±k σ)
  - `medianCurves`: 9레짐 × 8구간 중위 Δbp + 셀별 표본수 + 소스 레벨(cell/row/global)
  - `meta`: 산출 기간, k값, 산출일
- repo 관례(정적 JSON·fetch 부재, `<script src="data/*.js">` 전역 할당 — file:// CORS
  회피)에 정렬. JSON 파일·fetch 사용 금지.

**방법:**
- 일별 롤링 1개월 변화로 표본 확보 (중첩 윈도우 — 자기상관 존재하나 중위값 추정 유효)
- 각 관측을 (금리 방향, 스프레드 방향) 9셀로 분류 (밴드는 위 §2 값 사용 — 순환 의존
  방지 위해 밴드 먼저 확정 후 셀 분류)
- 셀별 구간 Δ 중위값 산출
- **계층적 폴백**: 셀 표본 < 30 → 행(금리 방향) 주변부 중위값 → 전체 중위값.
  결과 JSON에 소스 레벨(cell/row/global) 기록, UI 소스 뱃지 표시.

## 4. 데이터 모델 (확정 — Phase 0 조사 반영)

**저장 매체 2층:**

| 층 | 매체 | 용도 |
|---|---|---|
| 작업본 | localStorage (개인, 비커밋) | 입력 중인 미확정 값, 커브 세션 입력 |
| 확정본 | `data/rg-ledger.js` (커밋, 팀 공유) | 확정된 주간 판단 + 채점 레코드 |

**확정 흐름 (static 페이지는 repo에 쓸 수 없으므로 스니펫 방식):**
```
입력 UI (localStorage 작업본)
  → [확정] 버튼 → 복사용 JS 스니펫 생성 (레코드 1건)
  → data/rg-ledger.js 에 붙여넣기 → 커밋
```
- 채점도 동일: 실현값 입력 → 채점 결과 스니펫 → 원장 append → 커밋.

**data/rg-ledger.js 구조:**
```js
window.RG_LEDGER = {
  judgments: {
    "2026-W29": { probs: {...}, curves: {...}, sectors: {...},
                  baseline: {...}, confirmedAt: "..." },
    ...
  },
  scores: {
    "2026-W29": { realized: {...}, metrics: {...}, scoredAt: "..." },
    ...
  }
};
```

- **재확정**: 같은 주차 재확정 시 최신값으로 교체 + `confirmedAt` 갱신 (이력 미보존 —
  OO 패턴 정렬).
- **채점 불변**: 채점 완료 주차의 judgment/score 항목은 수정하지 않는다 —
  코드 강제가 아닌 **원장 운영 규칙(관례)**. 오입력 시 score 항목 삭제 후 재채점.
- 민평 수익률 원본은 어느 층에도 저장 금지 (§0.3). baseline에는 판정용 파생값만.

## 7. 확정된 설계 결정 이력 (토론 요약)

1. 판정 방식: 단일 셀 아닌 **9셀 확률 분포** (B안) — Brier 채점 가능
2. 호라이즌: **1개월 단일**, 주간 갱신 (중첩 창 순차 채점)
3. 보합밴드: **σ 기반**, 입력·채점 동일 적용
4. 노출 수위: **숫자만 입력, 근거 미기록**. 플레이북은 이론 수위, 버킷 매핑 제외
5. 롤다운: **v1 평행 + v2 내전망 + w 혼합**. 캐리는 현재 수익률로 확정,
   v1/v2 차이는 착지커브(커브이동 성분)만
6. v2 입력: **시나리오별 24칸 + 기본값** (역사 조건부 + 전주 이월 2층)
7. 기본값 조건화: 단순 금리 방향 아닌 **9레짐 조건부**, RG-1 스프레드 확률로 가중
8. 섹터: 6섹터 **간단형**(확률 3개씩), 회사채는 RG-1과 입력 공유
9. 확률 UX: **수동 + 합계 경고 + 원클릭 정규화** (자동 비례조정 없음)
